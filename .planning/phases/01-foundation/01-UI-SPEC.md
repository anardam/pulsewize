## Spacing Scale

Source: `@tesserix/tokens` spacing scale (8-point base).

| Token | @tesserix/tokens key | Value | Tailwind equivalent | Usage |
|-------|---------------------|-------|---------------------|-------|
| xs | `spacing[1]` | 4px | `gap-1` / `p-1` | Icon-to-label gaps, tight inline padding |
| sm | `spacing[2]` | 8px | `gap-2` / `p-2` | Compact form field spacing, chip padding |
| md | `spacing[4]` | 16px | `gap-4` / `p-4` | Default element spacing, card internal padding |
| lg | `spacing[6]` | 24px | `gap-6` / `p-6` | Section padding, form group gaps |
| xl | `spacing[8]` | 32px | `gap-8` / `p-8` | Layout section gaps, navbar height padding |
| 2xl | `spacing[12]` | 48px | `gap-12` / `py-12` | Major section breaks on landing page |
| 3xl | `spacing[16]` | 64px | `gap-16` / `py-16` | Hero and page-level vertical spacing |

**Exceptions:**
- Touch targets (buttons, avatar, nav links): minimum 44px height — use Tesserix `Button` default height (44px on `size="default"`)
- Navbar height: 64px — set via `TopNav` component's built-in height token
- Avatar size: 32px — use `Avatar` with `size="sm"` prop in `UserMenu`

## Typography

Source: `@tesserix/tokens` type scale. Use Tesserix `Heading` and `Text` components where possible; fall back to Tailwind classes only for one-off overrides.

| Role | Size | Weight | Line Height | Component / Class |
|------|------|--------|-------------|-------------------|
| xs label | 12px | 400 (regular) | 1.4 | `<Text size="xs">` |
| Body / forms | 14px | 400 (regular) | 1.5 | `<Text size="sm">` |
| Subheading | 20px | 600 (semibold) | 1.3 | `<Heading size="sm">` or `<Text size="xl" weight="semibold">` |
| Display / hero | 36px | 600 (semibold) | 1.2 | `<Heading size="xl">` |

**Weights in use:** 400 (regular) and 600 (semibold) — Geist variable font supports both weights via Tesserix token bindings.

**Mono usage:** `font-mono` (`--font-geist-mono`) reserved for code snippets, metric numbers in report cards, and API-related labels only. Use `<Text family="mono">` when the Tesserix `Text` component supports it; otherwise apply `className="font-mono"`.

**Gradient text:** Apply `.gradient-text` utility class (already in `globals.css`) to hero headlines and feature highlight words. Gradient: `135deg, #a855f7 → #ec4899 → #f97316`. The `HeroSection` component accepts a `gradientText` prop — use this instead of the raw class when available.

## Color

Source: `@tesserix/tokens` color scale + existing `globals.css` dark palette. The `violet` Tesserix theme maps `--primary` to violet-500 (`#a855f7`), which aligns exactly with the existing accent.

### Base Palette

| Role | Value | @tesserix/tokens ref | Usage |
|------|-------|----------------------|-------|
| Dominant (60%) | `#0a0a0f` | `colors.gray[950]` (closest) — override via CSS var `--background` | Page background, body — all pages |
| Secondary (30%) | `#111118` | `colors.gray[900]` (override) — CSS var `--card` | Cards, navbar, sidebar panels |
| Elevated surface | `#1a1a2e` | CSS var `--secondary` | Inputs, code blocks, hover states on cards |
| Border | `rgba(255,255,255,0.08)` | CSS var `--border` | All card borders, input borders |
| Text primary | `#ededed` | CSS var `--foreground` | Body copy, headings |
| Text secondary | `rgba(237,237,237,0.6)` | CSS var `--muted-foreground` | Labels, captions, placeholders |
| Text muted | `rgba(237,237,237,0.35)` | `text-muted-foreground/60` | Disabled states, footnotes |

### Accent

| Token | Value | Reserved For |
|-------|-------|-------------|
| Accent primary | `#a855f7` (violet-500) | Gradient start point, `--primary` Tesserix token |
| Accent end | `#ec4899` (pink-500) | Gradient end point, `.gradient-text` |
| Accent gradient | `linear-gradient(135deg, #a855f7, #ec4899)` | Primary CTA button background, active nav indicator, scrollbar thumb, hero words, card glow on hover, focus ring glow |

**Accent is NOT used on:** secondary buttons, body text, borders (except glow), icons in a resting state, form labels, error messages.

### Semantic

| Role | @tesserix/tokens ref | Value | Usage |
|------|---------------------|-------|-------|
| Destructive | `colors.red[500]` / `--destructive` | `#ef4444` | Delete account, remove data actions only |
| Success | `colors.green[500]` | `#22c55e` | Analysis complete toast, usage under limit indicator |
| Warning | `colors.amber[500]` | `#f59e0b` | Usage near limit (80%+) indicator |

### Tesserix Theme Token Overrides

The `violet` theme handles `--primary` correctly. Override remaining surface colors in `globals.css` inside `.dark` to match the existing dark palette:

```css
.dark {
  --background: 240 14% 4%;       /* #0a0a0f */
  --foreground: 0 0% 93%;         /* #ededed */
  --card: 240 14% 7%;             /* #111118 */
  --card-foreground: 0 0% 93%;
  --popover: 240 14% 7%;
  --popover-foreground: 0 0% 93%;
  --primary: 271 91% 65%;         /* #a855f7 — matches Tesserix violet theme */
  --primary-foreground: 0 0% 100%;
  --secondary: 240 14% 11%;       /* #1a1a2e */
  --secondary-foreground: 0 0% 93%;
  --muted: 240 14% 11%;
  --muted-foreground: 0 0% 60%;
  --accent: 271 91% 65%;
  --accent-foreground: 0 0% 100%;
  --destructive: 0 84% 60%;       /* #ef4444 */
  --destructive-foreground: 0 0% 100%;
  --border: 240 14% 15%;
  --input: 240 14% 11%;
  --ring: 271 91% 65%;
  --radius: 0.5rem;
}
```

## Copywriting Contract

### Global Tone

Casual and friendly. Use contractions ("you're", "we'll"). Avoid corporate jargon. Second-person ("you", "your"). Sentence case throughout (not Title Case) except for proper nouns and page titles.

### Primary CTAs

| Context | Label |
|---------|-------|
| Landing hero, above the fold | Get started |
| Landing secondary CTA | See how it works |
| Navbar (unauthenticated) | Get started |
| Signup page submit | Create account |
| Login page submit | Log in |
| Google OAuth button (both pages) | Continue with Google |
| Analyze page trigger | Analyze profile |
| Analyze page re-run | Run again |

### Auth Page Link Copy

| Location | Copy |
|----------|------|
| Below signup form | Already have an account? Log in |
| Below login form | Don't have an account? Get started |
| Login page — forgot password (future phase) | Omit from Phase 1 |

### Empty States

| Page | Heading | Body |
|------|---------|------|
| Dashboard — no reports yet | No analyses yet | Run your first analysis to see insights here. |
| Reports — no saved reports | Nothing saved yet | Analyses you run will appear here. |
| Reports — filtered, no results | No matches | Try adjusting your search or clearing the filter. |

### Error States

| Trigger | Copy |
|---------|------|
| Login — invalid credentials | Incorrect email or password. Please try again. |
| Login — unverified email | Check your inbox to verify your email before logging in. |
| Signup — email already in use | An account with this email already exists. Log in instead. |
| Signup — weak password | Password must be at least 8 characters. |
| Analyze — scraper failed, no data | We couldn't fetch that profile. Check the username and try again. |
| Analyze — usage limit reached (free) | You've used all 3 free analyses this month. Upgrade to continue. |
| Analyze — network/API error | Something went wrong. Please try again in a moment. |
| Generic server error | Something went wrong on our end. Try refreshing the page. |

### Destructive Actions

| Action | Trigger Label | Confirmation Copy |
|--------|--------------|-------------------|
| Delete saved report | Delete report | Delete this report? This can't be undone. |

**Confirmation pattern:** Tesserix `AlertDialog` (exported from `@tesserix/web`) — title matches action name, body is the confirmation copy above, confirm button uses `variant="destructive"`, cancel button is secondary.

### Usage / Limit Messaging

| State | Copy |
|-------|------|
| Free plan, analyses remaining | {N} of 3 free analyses used this month |
| Free plan, limit reached | You've reached your free limit for this month |
| Pro plan | Unlimited analyses |

## Focus & Accessibility

- All interactive elements must have visible focus ring. Tesserix `violet` theme maps `--ring` to violet-500 (`#a855f7`). No custom focus overrides needed.
- Color contrast: all text on dark backgrounds must meet WCAG AA (4.5:1 for body, 3:1 for large text). Primary text `#ededed` on `#0a0a0f` = 16.5:1 (passes AAA).
- All form inputs must have associated `<Form.Label>` — no placeholder-only labeling.
- `<UserMenu>` must pass `aria-label` of the user's email or "Your account" to the trigger `<Avatar>`.
- Destructive `<AlertDialog>` must be keyboard-navigable — Tesserix `AlertDialog` handles this via Radix internally.
- `<EmptyState>` action buttons must be keyboard-reachable with visible focus.
